# Bitstream — AI Context Document

## What Bitstream Is

Bitstream is a self-custodial Bitcoin cashflow engine built on Mezo. It allows users to lock BTC as collateral, mint MUSD (Mezo's native stablecoin), and automate recurring payments to any recipient or service — without ever selling their Bitcoin. The core thesis is: your Bitcoin should work for you passively, the way a paycheck works, not sit idle while you manually manage liquidity.

The product is being built for the Mezo hackathon under the MUSD Supernormal dApps track and the Bank on Bitcoin track.

## The Three-Layer Architecture

### Layer 1 — Capital Layer (Mezo-native)
- Users connect via Mezo Passport (the identity/wallet layer on Mezo)
- BTC collateral is deposited into the BitStreamVault smart contract
- MUSD is minted or accessed against that collateral
- The vault tracks collateral ratio at all times and enforces a minimum safety threshold
- If the ratio drops below threshold, payments pause and alerts fire
- All vault actions are on-chain, self-custodial, and auditable

### Layer 2 — Automation Layer (Agentic Treasury Manager)
- An off-chain Node.js + TypeScript agent runs continuously
- Uses BullMQ for job scheduling — checks every hour for upcoming payments
- Before executing any payment, it verifies: collateral ratio OK, MUSD balance sufficient, spending cap not exceeded
- Decides the payment route: direct MUSD transfer OR x402 payment to a service endpoint
- Emits alerts and logs everything — execution history is surfaced in the dashboard
- Can be thought of as the "brain" of the system — it makes financial decisions autonomously within user-defined rules

### Layer 3 — Payment Layer (x402)
- x402 is an HTTP-native payment protocol where services return 402 Payment Required with payment details
- Bitstream's agent uses x402-fetch to handle the full loop: call endpoint → receive 402 → pay in MUSD → retry → receive result
- This means Bitstream can pay internet-native services (APIs, SaaS tools, data feeds, merchant endpoints) directly in MUSD
- No bank accounts, no fiat rails, no checkout flows — pure programmable internet money
- Smart fallback: if x402 endpoint is unavailable, the agent falls back to direct MUSD wallet transfer
- The backend also exposes its own x402 endpoints (e.g. /api/premium/forecast) using x402-express middleware

## Smart Contracts

### BitStreamVault.sol
- Core vault contract
- Functions: depositCollateral(), mintMUSD(), schedulePayment(), executePayment(), checkCollateralRatio(), pauseVault()
- Events: Deposited, PaymentScheduled, PaymentExecuted, CollateralAlert, VaultPaused
- Access control: OpenZeppelin Ownable + Pausable
- Every external function checks: not paused, caller authorized, collateral ratio safe

### PaymentScheduler.sol
- Manages recurring payment schedules
- Parameters per schedule: recipient address or x402 endpoint URL, amount (MUSD), interval (seconds), max spending cap, active/paused flag
- Enforces spending caps per recipient and globally per vault

### CollateralGuard.sol
- Standalone module for collateral ratio logic
- Threshold: default 150% (configurable per vault)
- Called before every payment execution

## Backend / Agent

**Runtime:** Node.js + TypeScript  
**Framework:** Express.js  
**Job Queue:** BullMQ (Redis-backed)  
**Contract interaction:** ethers.js / viem  
**Validation:** Zod  
**x402:** x402-fetch (client), x402-express (server middleware)

### Key Files
- `scheduler.ts` — BullMQ cron job, runs every hour, iterates all active vaults
- `paymentExecutor.ts` — decides route (x402 vs direct), executes, logs result
- `vaultMonitor.ts` — polls collateral ratios, fires alerts if below threshold
- `x402Client.ts` — wraps x402Fetch with wallet signing and retry logic
- `x402Middleware.ts` — exposes paid API endpoints on the backend (e.g. cashflow forecast)

### Agent Decision Loop (every hour)
1. Fetch all active vaults from contract
2. For each vault: check collateral ratio → if low, emit alert and skip
3. Check upcoming payments due in the next window
4. Check MUSD balance ≥ payment amount
5. Check spending cap not exceeded
6. Execute: if payment.type === 'x402' → x402Fetch; else → contract.executePayment()
7. Log result with txHash, timestamp, endpoint, amount

## Key Product Rules (never violate these)

1. All user funds are self-custodial at all times — the vault never takes custody
2. Every payment checks collateral ratio before executing — no exceptions
3. x402 always has a fallback to direct MUSD transfer if the endpoint is unavailable
4. Mezo Passport is the only wallet/identity layer — no MetaMask-only flows
5. MUSD is the sole spending and payout currency throughout the entire system
6. Emergency pause must work on every vault function — one call stops everything safely
7. Spending caps are enforced both per-recipient and globally per vault

## Live App

Deploying to Vercel (frontend) + Railway/Render (backend agent),## Idle Asset Engine + Swap Layer

Beyond scheduled payments, Bitstream also functions as an intelligent idle asset optimizer. If a user holds assets sitting dormant — whether BTC not yet deployed, ETH, stablecoins, or other tokens — Bitstream's agent detects the idle capital and puts it to work automatically.

### How It Works

1. **Asset Detection** — The agent scans connected wallet balances and identifies assets sitting idle (below a user-defined activity threshold)
2. **Auto-Swap to BTC** — Idle non-BTC assets are automatically swapped into BTC via integrated DEX routing, consolidating the user's capital into Bitcoin as the base reserve asset
3. **Deploy into Mezo** — The swapped BTC is then routed into Mezo, deposited as collateral in the BitStreamVault, and used to mint additional MUSD — expanding the user's spending and payment capacity without any manual action
4. **Yield + Cashflow Loop** — The newly minted MUSD feeds back into the scheduling engine, increasing available cashflow for recurring payments or being allocated into approved yield strategies
5. **Agentic x402 Payments** — Once capital is deployed, the agent also autonomously executes x402 payments on behalf of the user — paying for API access, SaaS tools, data feeds, or any x402-compatible service endpoint in MUSD, fully autonomously within user-defined spending policies

### The Full Autonomous Loop
Idle Assets (any token) ↓ [agent detects + swaps] BTC ↓ [agent deposits into Mezo] BTC Collateral → MUSD Minted ↓ [vault manages] Scheduled Payments + x402 Agent Payments ↓ [recurring, automatic, self-custodial] Bills / Subscriptions / APIs / Recipients paid



This makes Bitstream a fully closed-loop Bitcoin financial agent — it finds your idle money, converts it to your hardest asset, deploys it productively on Mezo, and uses the liquidity to run your financial life on autopilot.

### Key Agent Behaviors
- Never swaps assets without user-defined approval rules (token whitelist, max swap size, slippage tolerance)
- Monitors swap routes for best execution before committing
- All swaps, deposits, and x402 payments are logged with full audit trail in the dashboard
- Agent can be paused or restricted per-asset class at any time
- Self-healing: if a swap fails, the agent retries with adjusted slippage or flags for manual review
